## Rules for Agents

- **Always use `--json`** unless you are piping one value to another command — then `--quiet`.
- **`--dry-run` first** on any write or delete. Read the printed request, then rerun without `--dry-run`.
- **`--help` is ground truth.** If this skill shows a flag that `--help` does not, the skill is stale — trust `--help`.
- **One command per step.** Do not chain in a single shell string; the agent reads each JSON result before deciding the next call.
- **Failures are data.** Exit code 2 = usage error (fix the command). Exit code 1 = API error (read the JSON body for details).
