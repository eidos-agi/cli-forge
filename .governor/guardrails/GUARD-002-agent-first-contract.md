## Rule

Every CLI generated by cli-forge must satisfy all six rules of the
Agent-First CLI Contract. A scaffold that cannot satisfy all six is a
broken scaffold — fix it before shipping. These rules are not style
preferences; they are the preconditions for an agent to use the tool
at all.

## The Six Rules

1. **`--json` on every command.** Structured, parseable output is the
   default mode. Default ON when stdout is not a TTY. Agents do not
   parse pretty tables.

2. **Non-interactive.** No `input()`, no `prompt()`, no wizards, no
   spinners, no color codes. Missing required args produce a fast error,
   never a prompt.

3. **Deterministic.** Same inputs produce same outputs. No UUIDs,
   timestamps, or random content in output unless the command is
   explicitly generating them. Agents rely on reproducibility to reason.

4. **Fast failure with actionable errors.** Exit 2 on usage errors with
   a message that includes the fix ("Error: --project-id is required.
   Example: mycli foo --project-id abc123"). No stack traces by default.

5. **`--dry-run` on destructive operations.** Any command that writes,
   deletes, or charges money must support `--dry-run`, which prints the
   request that would be made and exits 0.

6. **`--help` is the schema.** Every flag, type, default, and one
   realistic example per command. This is what replaces the MCP tool
   description in the agent's context.

## Why

MCPs give agents structured schemas for free. Taking the schema away
without replacing it with something equally structured leaves the agent
worse off. The six rules are how we rebuild that structure in a CLI.

Violating any one rule turns the generated CLI into a human-first tool
that agents can use only with heroic workarounds — which defeats the
entire purpose of cli-forge.

## Violation Examples

- A command that prompts with `Press Y to continue` (violates 2 and 5)
- Output with ANSI color codes even when `--json` is set (violates 1, 2)
- A list command that returns `Found 3 items: foo, bar, baz` in prose
  (violates 1)
- A `delete` command with no `--dry-run` flag (violates 5)
- A `--help` that says "see documentation for flags" (violates 6)
- A command that raises a Python traceback on a missing flag (violates 4)
- A command that includes `"timestamp": "..."` in every JSON response
  (violates 3 unless the timestamp is the actual payload)

## Enforcement

`/cli-forge-convert` runs a contract check as its final step and refuses
to emit a scaffold that fails any rule. Tests generated alongside the CLI
verify rules 1, 2, 4, 5, and 6 at build time.
